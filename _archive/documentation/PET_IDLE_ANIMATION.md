# Idle-анимация питомцев на пьедесталах

## Обзор

Питомцы на пьедесталах плавно покачивают головой благодаря GPU-шейдеру (PostFXPipeline). Ноги остаются неподвижными, движется только верхняя часть спрайта.

## Почему шейдер, а не tweens

В проекте включён `roundPixels: true` (файл `src/game/main.ts:13`), который нужен для чёткости pixel-art спрайтов. Однако он вызывает jitter (дёрганье) при любых мелких tween-трансформациях: rotation, sub-pixel translate, micro-scale. Шейдер работает на GPU после всех Phaser-расчётов, поэтому `roundPixels` на него не влияет.

## Ключевые файлы

| Файл | Что делает |
|------|-----------|
| `src/ui/IdleWobbleFX.ts` | PostFX pipeline с фрагментным GLSL-шейдером |
| `src/scenes/BootScene.ts` | Регистрация pipeline при старте игры |
| `src/ui/CenterStage.ts` | Применение pipeline к спрайтам питомцев |
| `src/core/config.ts` | Позиции пьедесталов (PEDESTAL) |

## Как работает шейдер

Фрагментный шейдер смещает UV-координаты текстуры по горизонтали (sway):

```glsl
float f = uv.y * uv.y;
float factor = f * f * uv.y;  // y^5 — крутая кривая
uv.x += sin(uTime * 2.8) * 0.017 * factor;
```

- `uv.y = 0` (ноги) — `factor = 0`, нет движения
- `uv.y = 0.5` (середина) — `factor = 0.03`, почти ничего
- `uv.y = 1.0` (голова) — `factor = 1.0`, максимальное покачивание

Степень `y^5` обеспечивает, что движется только верхняя ~25% спрайта.

## Параметры для тюнинга

В `src/ui/IdleWobbleFX.ts`:

- **Амплитуда**: `0.017` — множитель UV-смещения. Больше = сильнее качание
- **Скорость**: `2.8` — частота синусоиды. Больше = быстрее
- **Кривая**: `y^5` — степень falloff. Выше степень = движение ещё более сконцентрировано на макушке

В `src/ui/CenterStage.ts`:

- **Фазовый сдвиг**: `i * 1.5` — каждый пет качается с задержкой, чтобы не синхронно

## Привязка к пяточкам

Спрайты питомцев используют `setOrigin(0.5, 1)` — нижний центр. Это означает, что координата Y пьедестала определяет, где стоят ноги. Любой питомец, независимо от высоты, стоит ногами на одном уровне пьедестала.

Позиции пьедесталов настраиваются в `src/core/config.ts` → `PEDESTAL`.

Текст (имя + шанс) позиционируется динамически от верха изображения (`slot.image.y - slot.image.displayHeight`), а не от фиксированной точки.
